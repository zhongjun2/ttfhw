### Qwen2-0.5B 推理 —— 完整时间线

本阶段共经历 **5 次运行，4 次失败**，每次均需重新创建 venv 并安装依赖。

#### 第1次运行（13:03）— 失败

| 步骤 | 耗时 | 结果 |
|------|------|------|
| pip 安装（torch + modelscope + transformers） | **535s（≈9分钟）** | ✅ 安装完成（含超时重试） |
| pip 安装 torch-npu | 包含在上述时间内 | ✅ |
| 模型下载 | 3s | ❌ 失败 |
| 推理 | — | 未执行 |
| **小计** | **≈538s** | ❌ |

**失败原因：** torch-npu 安装顺序错误——torch-npu 在 modelscope 之前安装，导致 modelscope CLI 启动时崩溃（`torch_npu.npu` 初始化失败，libhccl.so 缺失）。

#### 第2次运行（13:14）— 失败

| 步骤 | 耗时 | 结果 |
|------|------|------|
| pip 安装 | 88s | ✅ |
| **模型首次下载（Qwen2-0.5B 953 MB）** | **2012s（≈33.5分钟）** | ✅ 下载完成 |
| pip 安装 torch-npu（下载后） | 含在安装时间内 | ✅ |
| 推理 | 3s | ❌ 崩溃 |
| **小计** | **≈2103s（≈35分钟）** | ❌ |

**失败原因：** 推理脚本中虽用 try/except 捕获 `import torch_npu`，但 `import torch` 时会通过 entrypoint 机制**再次自动加载** torch_npu 后端，绕过了 sys.modules 中的占位符：
```
RuntimeError: Failed to load the backend extension: torch_npu.
You can disable extension auto-loading with TORCH_DEVICE_BACKEND_AUTOLOAD=0
```

#### 第3次运行（13:50）— 失败

| 步骤 | 耗时 | 结果 |
|------|------|------|
| pip 安装 | 86s | ✅ |
| 模型下载（缓存） | 9s | ✅ |
| 推理 | 4s | ❌ |
| **小计** | **≈99s** | ❌ |

**失败原因：** 在 sys.modules 中注入了 dummy module，但 dummy module 的 `__spec__` 为 None。transformers 在导入时调用 `importlib.util.find_spec('torch_npu')` 报错：
```
ValueError: torch_npu.__spec__ is None
```

#### 第4次运行（13:55）— 失败

| 步骤 | 耗时 | 结果 |
|------|------|------|
| pip 安装 | 84s | ✅ |
| 模型下载（缓存） | 7s | ✅ |
| 推理 | 4s | ❌ |
| **小计** | **≈95s** | ❌ |

**失败原因：** 修复了 `__spec__` 问题，但推理脚本中 `AutoModelForCausalLM.from_pretrained(path, device_map='cpu')` 报错：
```
ValueError: Using a `device_map` ... requires `accelerate`.
```
transformers 5.x 将 device_map 处理移到 accelerate 包，需单独安装。

#### 第5次运行（14:03）— 成功 ✅

| 步骤 | 耗时 | 结果 |
|------|------|------|
| pip 安装（含 accelerate） | 83s | ✅ |
| 模型下载（缓存） | 6s | ✅ |
| 推理 | 6s | ✅ |
| **小计** | **≈95s（缓存）** | ✅ |

**推理输出：** `你好，我最近感觉很`

**最终修复方案（3 处修改）：**
1. `TORCH_DEVICE_BACKEND_AUTOLOAD=0` — 禁止 torch 通过 entrypoint 自动加载 torch_npu 后端
2. `dummy.__spec__ = importlib.machinery.ModuleSpec(...)` — 给占位模块设置合法 spec，避免 transformers find_spec 报 ValueError
3. 依赖列表加入 `accelerate` — 满足 transformers 5.x `device_map` 使用前提
