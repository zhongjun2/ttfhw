## 一、会话目标

用户希望针对一名新用户从零开始使用 CANN 的完整路径构建一套验证方案：

1. 在 Google 搜索 CANN 相关信息
2. 根据搜索到的信息下载并安装 CANN
3. 根据社区文档完成示例场景（Quick Start + 模型推理）

要求：
- 所有步骤逻辑来自搜索到的社区文档，不允许自行发明
- 每步操作注明出处（URL + 文章 + 段落）
- 输出包含了解 / 获取 / 使用三阶段耗时、操作步骤和断点问题的中文评估报告

## 四、评估发现（来自 reports/2026-03-27.md）

### 断点汇总（7 个）

| 编号 | 阶段 | 级别 | 核心问题 | 对 CANN 产品的改进价值 |
|------|------|------|---------|----------------------|
| P1-01 | 了解 | P1 | WebFetch 工具网络受限（测试环境问题） | — |
| P1-02 | 获取 | P1 | AscendHub 域名 `ascendhub.huawei.com` DNS 不可达 | 镜像仓库可达性需改善，或文档提供备用公开镜像 |
| P1-03 | 获取 | P1 | SWR 镜像需身份认证，文档未提供匿名拉取路径 | 文档应补充：匿名用户如何获取镜像（如 Docker Hub `ascendai/cann`） |
| P1-05 | 使用 | P1 | Quick Start helloworld 示例所需模型文件不在容器内 | 文档应补充模型文件获取步骤，或容器中预置示例数据 |
| P1-06 | 使用 | P1 | 无 NPU 时 `device_map='auto'` 不可用（CPU 环境） | 文档可补充 CPU-only 测试路径说明 |
| P2-04 | 获取 | P2 | Docker 组权限未刷新（session 未重启） | — |
| P2-07 | 使用 | P2 | 模型缓存命中，未实测网络下载 | — |

### 易用性关键发现

1. **SEO 可发现性（本次）**：搜索 `CANN 昇腾 安装`，官方 hiascend.com 排名**第 1 条**（与上一次 cann-eval 评估结果不同，上次未出现在前 10 条）。

2. **镜像获取是最大断点**：官方文档提供的两个镜像源（AscendHub / SWR）均无法对新用户直接可用（DNS 或认证），实际可用的公开镜像（`ascendai/cann:latest`）未在官方文档中出现。

3. **Quick Start 示例不完整**：文档展示了 ATC 模型转换命令，但容器中没有示例所需的模型文件，也未说明如何获取，新用户会在第一步就卡住。

4. **Qwen2 推理可跑通**：在无 NPU 的 CPU 环境下，完整推理流程（pip install → 模型加载 → generate）可以成功执行，耗时合理（pip 96s + 推理 14s，不含模型下载）。
