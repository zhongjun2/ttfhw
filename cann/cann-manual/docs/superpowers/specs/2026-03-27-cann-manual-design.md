# CANN 手动评估协议 — 架构设计规格

**版本：** v1.0
**创建日期：** 2026-03-27
**状态：** 正式

---

## 一、目标

由 Claude 扮演一名新用户，通过 Google 搜索找到 CANN 官方文档，严格按照文档描述的步骤完成安装和模型推理，记录全程耗时与断点问题，输出中文结构化评估报告。

**核心约束：**
- 所有操作步骤必须来源于搜索结果中找到的网页内容，不得自行发明或修改逻辑
- 每步操作必须注明出处：URL + 文章标题 + 具体段落/章节
- 执行命令必须是文档里原文描述的命令（允许补充必要的环境参数，但需注明）

---


## 三、三阶段划分

| 阶段 | 起点 | 终点 | 核心问题 |
|------|------|------|---------|
| **了解** | 开始第一次 Google 搜索 | 找到所有目标链接并验证可访问 | 官方资料能不能被搜到？链接有效吗？ |
| **获取** | 从文档提取安装命令 | CANN 环境就绪（`atc --help` 正常） | Docker 方式需要多长时间？有哪些坑？ |
| **使用** | 容器/环境可用 | Qwen2 推理输出结果 | 工具链可用？示例能跑通？ |

计时规则：每个阶段用 `date +%s` 记录 `start_time` / `end_time`，报告中换算为分钟。

---

## 四、各阶段详细步骤

### 4.1 阶段一：了解（Learn）

| 步骤 | 操作 | 工具 |
|------|------|------|
| L1 | 搜索 `"CANN 昇腾 安装"` | WebSearch |
| L2 | 在结果中找 `hiascend.com` 官方链接，记录排名（第几条） | — |
| L3 | 打开官方链接，找到安装文档入口 | WebFetch |
| L4 | 在文档网站内找 Quick Start 链接 | WebFetch |
| L5 | 搜索 `"Qwen2 CANN 昇腾 推理"` 找模型部署文档 | WebSearch |
| L6 | 逐一验证所有找到的链接是否可访问（HTTP 状态码） | WebFetch |

**记录指标：**
- 搜索耗时
- hiascend.com 在搜索结果中的排名（null = 未找到）
- 找到的链接列表（URL + 标题 + 可访问性）
- Quick Start 链接是否找到
- Qwen2 部署文档是否找到

### 4.2 阶段二：获取（Get）

仅测 Docker 方式（.run 包需要 root 权限，当前环境受限）。

| 步骤 | 操作 | 工具 |
|------|------|------|
| G1 | 从 Quick Start 文档中提取 docker pull 命令及镜像名 | WebFetch |
| G2 | 执行文档中的 `docker pull` 命令 | Bash |
| G3 | 按文档说明启动容器，执行 `atc --help` 验证工具链 | Bash |

**记录指标：**
- 从哪篇文档哪段提取了 docker 命令
- 镜像下载耗时、镜像大小
- atc 工具链是否可用

### 4.3 阶段三：使用（Use）

| 步骤 | 操作 | 工具 |
|------|------|------|
| U1 | 从 Quick Start 文档提取示例命令（如 atc 模型转换）并执行 | WebFetch + Bash |
| U2 | 从 Qwen2 部署文档提取依赖安装命令 | WebFetch |
| U3 | 按文档安装依赖（pip install）、下载模型 | Bash |
| U4 | 按文档执行推理命令，记录输出 | Bash |

**记录指标：**
- 每步来源文档
- pip 安装耗时
- 模型下载耗时、大小
- 推理是否成功、推理输出
- 遇到的所有断点

## 五、来源引用格式

每步操作后必须附一条来源注释：

```
来源：https://example.com/page → 《文章标题》→「具体段落或章节描述」
```

示例：
```
来源：https://www.hiascend.com/document/detail/zh/CANNCommunityEdition/80RC3alpha003/quickstart/quickstart/quickstart_18_0002.html → 《CANN 快速上手》→「获取 CANN 容器镜像」章节
```

## 八、执行方式

Claude 在单次对话中按本协议顺序执行：
1. 依次使用 `WebSearch` → `WebFetch` → `Bash`
2. 每步操作后立即记录来源注释和耗时
3. 遇到错误不跳过，先尝试按文档描述的方法解决，解决不了则记为断点后继续
4. 全部阶段完成后，将记录汇总为 `reports/YYYY-MM-DD.md`

**不覆盖范围：**
- .run 包裸机安装（需 root）
- 物理 NPU 推理性能指标
- 多机多卡分布式
